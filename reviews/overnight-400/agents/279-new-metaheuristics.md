# 279 — new-metaheuristics (Tabu / ALNS / GRASP / ILS / VNS / LAHC / Path-Relinking)

## Headline
reality v0.10.0 ships ZERO combinatorial-metaheuristic surface (only `optim/metaheuristic.go::SimulatedAnnealing` 93 LOC + `optim/genetic.go::GeneticAlgorithm` 178 LOC, both real-coded `[]float64`-only with no neighborhood-move abstraction); a 9-primitive `optim/metah/` sub-package (~1850 LOC, generic over Candidate type-parameter) is unblocked TODAY and is the natural feasible-solution layer that pairs with slot-277 exact MIP B&B and slot-278 Lagrangian relaxation to complete the NP-hard combinatorial-optimisation pillar — TabuSearch + LAHC + ILS + path-relinking ship in ~600 LOC day-1 with zero blockers.

## Findings

- **F1. Existing metaheuristic surface is real-coded R^n only.** `optim/metaheuristic.go::SimulatedAnnealing(f, x0, neighbor, temp0, cooling, maxIter, rng)` (lines 38-93) takes `func([]float64) float64` and `neighbor func(current, out []float64)`; no abstraction for permutation, set, assignment, tree, or generic discrete candidates. `optim/genetic.go::GeneticAlgorithm` (lines 44-178) is hard-coded BLX-α real-coded with population init in `[-5, 5]^dim`. Neither composes with a TSP tour, a knapsack subset, a graph colouring, or a VRP routing schedule.
- **F2. Zero combinatorial-heuristic API exists.** Repo-wide grep on `Tabu|TabuSearch|ALNS|GRASP|IteratedLocal|VariableNeighborhood|VNS|GuidedLocal|LAHC|LateAcceptance|PathRelink|ScatterSearch|RuinRecreate|Aspiration` returns ZERO `*.go` source matches (only review-doc hits). Grep on `Neighborhood|NeighborMove|LocalSearch` returns ZERO `*.go` source matches. `Move`/`Aspiration`/`TabuList`/`Diversification`/`Intensification` keywords absent.
- **F3. Slot 102 already enumerated this gap as Tier-2/Tier-3.** `reviews/overnight-400/agents/102-optim-missing.md` lines 105-106 list T2.19 Tabu (Glover 1986) and T2.20 ALNS (Pisinger-Røpke 2007); lines 159-161 list T3.11 GRASP, T3.12 VNS (Hansen-Mladenović 1997), T3.13 ILS (Lourenço-Martin-Stützle 2003). Slot 279 is the dedicated execution slot for that backlog cluster.
- **F4. Slot 277 (NP-hard exact MIP) and slot 278 (Lagrangian relaxation) ship the *exact* and *bound* sides of NP-hard combinatorial optimisation; slot 279 owns the *feasible-heuristic* third leg.** All three converge on the same TSP / VRP / set-cover / vertex-cover / bin-packing / scheduling consumer surface. Slot 277-I13 TSP B&C uses lazy subtour-elimination cuts and benefits from a metaheuristic warm-start incumbent (industrial Concorde, CPLEX, Gurobi all do this); slot 278-R4 HeldKarpBound provides the LB; slot 279 provides the UB upper-bound incumbent that prunes the B&C tree by orders of magnitude.
- **F5. CMA-ES / DE / PSO / Bayesian-opt are NOT in the codebase yet** (assumed in prompt context) — grep on `func CMAES|DifferentialEvolution|ParticleSwarm|BayesianOpt` returns zero source matches. Slot 102 lists CMA-ES as T2.18, BO as T2.21. Slot 279 should NOT depend on those; ship combinatorial-heuristic layer independently.
- **F6. Existing `SimulatedAnnealing` is type-incompatible with combinatorial state.** Cooling schedule (geometric `temp *= cooling` line 89) is reusable, but the `[]float64`-shaped current/best/cand workspaces (lines 49-59) prevent reuse on a `[]int` permutation or a `[]bool` knapsack vector without an adapter. **Right move:** introduce `Candidate any` type-parameter via Go generics (Go 1.18+); `SimulatedAnnealing` stays as the real-coded specialisation, new `metah.SA[C any]` is the generic version. Slot 110 perf review noted CLAUDE.md rule-3 mandates zero-alloc hot paths — the generic `Move[C]` interface needs in-place `Apply(c)`/`Undo(c)` semantics, not value-copy semantics.
- **F7. Pure-Go zero-dep MIT-licensed status is unique moat.** Web-search 2024-2025 surface: `optaplanner-go` (does not exist; OptaPlanner is JVM/Java only); `nextmv-io/sdk/route` exists but is CGo + commercial license; `ALNS` Python (Wouda-Lan 2023) is the de-facto reference but Python; no Go ALNS / Tabu / GRASP libraries with MIT license + zero deps surfaced in search. Reality landing this would be the **only** pure-Go zero-dep MIT-licensed metaheuristic library — sibling-moat to slot 277 (only pure-Go MIP) and slot 278 (only pure-Go Lagrangian).
- **F8. TSP test instance is the canonical 3/3 mutual-cross-validation pin.** Same n=100 random Euclidean TSP under fixed seed: TabuSearch with 2-opt neighbourhood + tabu-tenure 25 ≡ ILS with 2-opt local search + double-bridge perturbation ≡ LAHC with 2-opt neighbourhood + history-length 100 — all should converge within 2% of Concorde-optimum within 10⁵ evaluations. Held-Karp 1-tree LB from slot-278-R4 provides the gap-to-optimal numerator. This single test pins R-MUTUAL-CROSS-VALIDATION 3/3.
- **F9. Operator-pool architecture is the ALNS / GRASP / hyper-heuristic generalisation.** Pisinger-Røpke 2007 ALNS uses 4-6 destroy operators (random-removal, worst-removal, Shaw-removal-by-relatedness, route-removal) + 3-4 repair operators (greedy-insertion, regret-2, regret-3, regret-k); each iteration picks one of each by roulette-wheel weighted by past success. **This same architecture covers** GRASP construction-phase RCL (restricted-candidate-list with α-greediness), Burke 2013 selection-hyper-heuristic, and adaptive-portfolio operator selection. **One generic `OperatorPool[C any]` struct serves all four** — major code-reuse win.
- **F10. LAHC is the cheapest pedagogical primitive in the entire combinatorial-metaheuristic canon.** Burke-Bykov 2008/2017 (PATAT-2008, EJOR 258:70) state full algorithm in ~30 LOC: maintain length-L history of fitness values, accept candidate iff `f(cand) ≤ history[iter mod L]` (or `f(cand) < f(current)`). Single parameter L (history length). No temperature, no cooling schedule, no acceptance probability. Beats SA on most timetabling/TSP benchmarks per published comparisons. **Right primitive to ship first** as the simplest non-trivial non-SA acceptance criterion.

## Concrete recommendations

1. **`optim/metah/move.go::Move[C any], Neighborhood[C any], Candidate[C any]` interfaces (~80 LOC, T0).** Generic over candidate type `C` (constrained `any` then refined). Core surface: `type Move[C any] interface { Apply(c C); Undo(c C); Delta(c C) float64 }`; `type Neighborhood[C any] interface { Random(c C, rng *rand.Rand) Move[C]; All(c C) []Move[C] }`; `type Candidate[C any] interface { Clone() C; Fitness() float64 }`. Apply/Undo enables zero-alloc hot path (CLAUDE.md rule-3). `Delta` enables incremental fitness for Tabu / LAHC / SA. Permutation 2-opt/3-opt/or-opt + knapsack flip + assignment-swap built-in concrete `Neighborhood` instances. **Unblocks all subsequent T1-T5 primitives.**

2. **`optim/metah/tabu.go::TabuSearch[C](init C, nbhd Neighborhood[C], cfg TabuCfg) (best C, bestF float64)` (~180 LOC, T1).** Glover 1986 *Comp Ops Res* 13:533 + Glover 1989/1990 ORSA-J Comput 1-2 (Tabu Search Parts 1-2). Move-attribute tabu list with hash-table tenure (FIFO ring buffer of attribute hashes); **aspiration criterion** = override tabu if move would yield new global best (Glover 1989 §4); **diversification** trigger when no improvement in last `D` iters → restart from best with perturbed tabu list; **intensification** = elite-solution archive of top-K with periodic local-search re-anchor. Reusable on TSP / set-cover / coloring. **Pin R-MUTUAL-CROSS-VALIDATION 3/3** with R3 LAHC + R4 ILS on n=100 random Euclidean TSP, gap ≤ 2% from Held-Karp 1-tree LB (slot-278-R4 dependency consumed not blocked — slot 278 ships independently).

3. **`optim/metah/lahc.go::LAHC[C](init C, nbhd Neighborhood[C], historyLen int, maxIter int, rng *rand.Rand) (best C, bestF float64)` (~80 LOC, T1, CHEAPEST).** Burke-Bykov 2017 *EJOR* 258:70 (and PATAT-2008 first-presentation). Single parameter `historyLen` L (typical 100-5000). Maintain `history [L]float64` initialised to `f(init)`; accept candidate iff `f(cand) <= history[iter mod L]` or `f(cand) < f(current)`; update `history[iter mod L] = f(current)` after each iteration. **No cooling schedule. No temperature. No probability.** Per-Burke-Bykov-EJOR-2017-Table-2: outperforms SA on 17 of 20 timetabling/TSP benchmarks at equal evaluation budget. **Cheapest pedagogical demo of acceptance-criterion-as-hyperparameter-free design.** Composes with R1 Move interface; ~80 LOC standalone.

4. **`optim/metah/ils.go::IteratedLocalSearch[C](init C, localSearch func(C) C, perturb func(C, *rand.Rand) C, accept func(prev, cand C) C, maxIter int) (best C, bestF float64)` (~100 LOC, T1).** Lourenço-Martin-Stützle 2003 *Handbook of Metaheuristics* ch.11. Outer loop: localSearch → perturb → localSearch → accept. **Composes** with any local-search inner kernel — including `TabuSearch` (R2), `LAHC` (R3), or future hill-climber. `perturb` for TSP = double-bridge 4-opt (Martin-Otto-Felten 1991); for knapsack = random k-flip; for coloring = Kempe-chain swap. `accept` strategies: better-only / Metropolis-acceptance / annealing-on-best-found. Dirt-cheap composition layer. **Same 3/3 pin** with R2 TabuSearch + R3 LAHC.

5. **`optim/metah/path_relink.go::PathRelink[C](src, dst C, attrDist func(C, C) int, makeMove func(C, C) Move[C]) (bestOnPath C, bestF float64)` (~120 LOC, T1).** Glover 1996/1997 *Tabu Search* (Glover-Laguna 1997 book) + Resende-Ribeiro 2005 *Path-Relinking Intensification Methods for Stochastic Local Search*. Walk in attribute-space from source to destination by greedy-improving moves that monotonically decrease symmetric-difference attribute distance, returning the best intermediate. **Intensification operator** for GRASP and scatter-search. Composes naturally with R1 Move + R8 GRASP elite-pool. ~120 LOC. **Standalone usable** with TabuSearch elite pool from R2.

6. **`optim/metah/lns.go::LargeNeighborhoodSearch[C](init C, destroy []DestroyOp[C], repair []RepairOp[C], cfg LNSCfg) (best C, bestF float64)` (~140 LOC, T2).** Shaw 1998 *CP-98* large-neighborhood-search original (uniform operator selection). Each iter: pick one destroy op (uniform), pick one repair op (uniform), apply destroy(current) → partial, apply repair(partial) → cand, accept with SA-style criterion. **Sub-template of R7 ALNS** without the adaptive layer; useful as pedagogical primitive and as fallback when adaptive weights have not warmed up. ~140 LOC composes with R1.

7. **`optim/metah/alns.go::AdaptiveLNS[C](init C, destroy []DestroyOp[C], repair []RepairOp[C], cfg ALNSCfg) (best C, bestF float64)` (~280 LOC, T2, MOAT).** Pisinger-Røpke 2007 *Comp Ops Res* 34:2403 (also DTU tech-report ORBIT 3154462; cf. Ropke-Pisinger 2006 *Transp Sci* 40:455 PDPTW). Adaptive operator-selection on top of R6 LNS: roulette-wheel weighted by `score[op] = sum_segment(σ_i × outcome_i)` with σ₁=33 new-best, σ₂=9 better-than-current, σ₃=13 accepted-worse, σ₄=0 rejected (Pisinger-Røpke 2007 Eq.10-12); weight reset per segment of N=100 iters; SA-Metropolis acceptance with start-temp `T₀ = -ω × f(init) / ln(0.5)` (ω=0.05) cooling `c=0.99975`. Built-in destroy ops (random-removal, worst-removal, Shaw-removal-by-relatedness with relatedness `R(i,j)=φ·d_ij + ψ·|t_i-t_j|`, route-removal) + repair ops (greedy-insertion, regret-k for k=2,3,4). **Reality's only ALNS** anywhere on the Go ecosystem (pure-Go zero-dep MIT moat; verified web-search-2025-no-pure-Go-ALNS-library). Cross-link: ships VRP / PDPTW / set-covering reference instance; with slot 277 owning lazy-cut B&C, ALNS + B&C is the OR-Tools / Gurobi / OptaPlanner-equivalent stack.

8. **`optim/metah/grasp.go::GRASP[C](nIter int, construct func(α float64, rng *rand.Rand) C, localSearch func(C) C, cfg GRASPCfg) (best C, bestF float64)` (~140 LOC, T3).** Feo-Resende 1989 *Op Res Lett* 8:67 + Feo-Resende 1995 *J Global Opt* 6:109 + Resende-Ribeiro 2010 *Greedy Randomized Adaptive Search Procedures: Advances and Applications*. Greedy-randomised construction phase: build solution element-by-element, at each step compute `candidates with cost ≤ c_min + α(c_max - c_min)` (the **RCL** — restricted candidate list), pick uniformly from RCL; α=0 pure-greedy, α=1 pure-random, α=0.3 typical. Each iter: construct → localSearch (e.g. R3 LAHC or R2 TabuSearch). Optional **Reactive-GRASP** Prais-Ribeiro 2000 *J Heur* 6:283 — adapts α by past-success roulette. Optional **path-relinking intensification** via R5 from elite pool. Generic over `construct` callable so consumers supply problem-specific greedy. ~140 LOC + reuse R5.

9. **`optim/metah/vns.go::VariableNeighborhoodSearch[C](init C, nbhds []Neighborhood[C], localSearch func(C, Neighborhood[C]) C, maxIter int, rng *rand.Rand) (best C, bestF float64)` (~120 LOC, T4).** Mladenović-Hansen 1997 *Comp Ops Res* 24:1097 + Hansen-Mladenović 2001 *EJOR* 130:449. Outer loop: **shake** (random move in `nbhds[k]` of widening structure) → **local search** (descend in `nbhds[1]`) → if improved restart with k=1 else k=k+1; reset k=1 on improvement, terminate when k > k_max. Variants: VND (deterministic shake-free), GVNS (general — local search itself uses VND), RVNS (no local search). Common k_max=5-10 for TSP (2-opt, 3-opt, or-opt, double-bridge, segment-reverse). Composes with R1 Neighborhood + R3 LAHC inner kernel. ~120 LOC.

10. **`optim/metah/gls.go::GuidedLocalSearch[C](init C, nbhd Neighborhood[C], features []Feature[C], maxIter int, λ float64) (best C, bestF float64)` (~140 LOC, T5, OPTIONAL).** Voudouris-Tsang 1997 *Tech-Rep CSM-247* + Voudouris-Tsang 2003 *Handbook of Metaheuristics* ch.7. Augmented objective: `f'(c) = f(c) + λ × Σᵢ pᵢ × Iᵢ(c)` where `Iᵢ(c)` = feature-i indicator and `pᵢ` = penalty. On local minimum: increase penalty on features with highest `cost_i / (1 + p_i)` utility ratio. Local search resumes on `f'`. λ controls penalty scale (typical λ = average-edge-cost × 0.3 for TSP). Pure feature-attribute method — needs no neighborhood structure beyond R1; works on any combinatorial problem with definable solution-attributes. ~140 LOC. Defer to T5 — less foundational than T1-T2 primitives.

11. **`optim/metah/scatter_search.go::ScatterSearch[C](popSize, refSize int, combine func(C, C, *rand.Rand) C, improve func(C) C, distance func(C, C) float64, maxGen int) (best C, bestF float64)` (~180 LOC, T5, OPTIONAL).** Glover 1977 *Decision Sci* 8:156 + Glover-Laguna-Martí 2000 *Control Cybern* 29:653 (chapter in *Optimization Software Class Libraries*). Reference-set-based EA with two halves: (a) high-quality solutions, (b) max-distance solutions for diversity. **Solution-combination** is deterministic linear-combination not stochastic crossover (key differentiator from GA); subset-generation enumerates 2-tuples / 3-tuples from refSet then improves. Pairs naturally with **path-relinking R5** as the combination operator. ~180 LOC. T5 — defer; less practical impact than R7 ALNS.

# 359 — research-correctness (test corpora reality should validate against)

## Headline
Adopt a multi-tier correctness corpus stack: mpmath/Arb for special functions, DLMF for tabulated reference, MPFR/CORE-MATH for ulp-bounded transcendentals, NIST CAVP for crypto, ACM TOMS/CALGO for algorithm-paper drivers — golden files generated offline, validated in CI.

## Test corpora

### 1. mpmath (BSD-3, Python)
- **License**: New BSD (3-clause). Fully redistributable; we can re-export tabulated values without contamination.
- **Coverage**: gamma/beta/zeta/Hurwitz, hypergeometric ₀F₁..ₚFq, Bessel J/Y/I/K, Airy, elliptic E/K/Π, polylogs, error fns, polygamma, theta — basically the entire `prob`/`signal`/`physics` analytic surface.
- **Format**: Python; emits arbitrary-precision strings via `mp.dps=50`. No native JSON bundle — generated on-demand.
- **Ingest**: write `tools/gen-mpmath-golden.py` that walks a YAML manifest (function, arg-grid, dps) and emits `testdata/<pkg>/<fn>.golden.json` with `{x, y_ref_str, y_ref_f64}`. Reality Go test parses string with `math/big.Float`, then compares `float64(y) == y_ref_f64` exactly or within fn-tolerance.
- **Latest**: 1.4.0 (2026-02-23). Stable API since 2008.

### 2. DLMF — NIST Digital Library of Mathematical Functions
- **License**: Public domain (US government work, NIST).
- **Coverage**: 36 chapters — gamma & beta (Ch.5), exponential integral (Ch.6), error/Dawson (Ch.7), incomplete gamma (Ch.8), Airy (Ch.9), Bessel (Ch.10), Struve (Ch.11), parabolic cylinder (Ch.12), confluent hypergeometric (Ch.13), Legendre (Ch.14), Gauss hypergeometric ₂F₁ (Ch.15), generalized hypergeometric (Ch.16), q-series (Ch.17), orthogonal polynomials (Ch.18), elliptic (Chs.19-23), zeta & related (Ch.25), combinatorial (Ch.26), Mathieu/Heun/Painlevé (Chs.28-32), Coulomb (Ch.33), 3j/6j/9j (Ch.34).
- **Format**: HTML with rendered MathML, plus PDF chapters. No machine-readable table dumps in the modern revision (A&S had numeric tables; DLMF removed them). Reference values appear inline as worked examples and "asymptotic to" anchors at boundary points.
- **Ingest**: scrape worked-example values into seed JSON; cross-check against mpmath at same precision. DLMF acts as the *spec*, mpmath as the *generator*.

### 3. ACM TOMS / CALGO (netlib.org/toms)
- **License**: ACM CALGO terms — non-commercial use unrestricted; commercial requires per-algorithm permission. 750+ algorithms, mostly Fortran.
- **Coverage**: Algorithms 493–1000+ (~1970s onward). Each TOMS "algorithm paper" ships an example driver, test problems, expected results. Notable: Alg 715 (SPECFUN, W. Cody special functions), Alg 644 (Bessel), Alg 757 (Cephes special-function complements), Alg 819 (AIRY), Alg 880 (Faddeeva), Alg 916 (Voigt), Alg 985 (incomplete gamma).
- **Format**: Fortran source + ASCII expected-output files (e.g. `algNNN/output`).
- **Ingest**: For algorithms whose drivers print `(x, f(x))` triples, parse output files into `testdata/calgo/algNNN.golden.txt`. License-clean for reference comparison even without redistribution. Cite source per file.

### 4. MPFR (LGPL-2.1+)
- **License**: LGPL — we cannot link or vendor, but we *can* observe outputs offline and embed numeric values as facts (numbers aren't copyrightable in US). Same theory as scipy's xsf usage.
- **Coverage**: Correctly-rounded transcendentals (sin/cos/tan/exp/log/pow/erf/gamma/zeta/Bessel J0/J1/Y0/Y1/Ai/Bi/Li2 and many more), correct over full IEEE-754 range, all 5 rounding modes. Ziv's strategy guarantees correct rounding to last bit.
- **Format**: C library; reference outputs via `mpfr_printf("%.17Rg")`.
- **Ingest**: golden generator (Go test build-tag `goldengen`, links libmpfr via cgo on Linux only) emits `testdata/mpfr/<fn>_RNDN.json`. Generator produces values; runtime uses pure-Go reality compared at <1 ulp.

### 5. Arb / FLINT (LGPL-2.1+)
- **License**: LGPL. Same observe-offline pattern as MPFR.
- **Coverage**: Rigorous interval (ball) arithmetic. Special functions with provable error bounds: ζ(s), Γ(z), hypergeometric ₀F₁/₁F₁/₂F₁, Hurwitz, Lerch, polylogs, theta, Weierstrass ℘, Riemann ξ, Dirichlet L, Stieltjes constants, Bernoulli numbers (Bn for n up to 10^6+ in seconds). Number theory: factor, isprime, modular forms.
- **Format**: C; emits `[mid ± rad]` ball strings. Mid → reference; rad → tolerance bound (rigorous).
- **Ingest**: Best for `prob` zeta/digamma, `crypto` Bernoulli/Bn factors, `combinatorics` partition p(n) (Arb computes p(10^9) in milliseconds — verifiable). Use radius as automatic tolerance.

### 6. SciPy xsref / xsf test corpus
- **License**: BSD-3 (SciPy). xsref repo: https://github.com/scipy/xsref.
- **Coverage**: Direct mirrors of every scipy.special function — Bessel, Struve, gamma family, error functions, hypergeometric, elliptic, Mathieu, parabolic cylinder. Generated by mpmath, vetted in production by NumPy/SciPy ecosystem (millions of users find regressions for us).
- **Format**: **Apache Parquet** files at `xsref/tables/scipy_special_tests/<fn>/{In_*,Out_*}.parquet`. Type-tagged (e.g. `In_d_d-d.parquet` = double-double → double).
- **Ingest**: Add `tools/import-xsref.py` that reads parquet via pyarrow and emits Go-friendly JSON. Cleanest single-source corpus available — already curated, type-shaped, edge-case populated.

### 7. Boost.Math test data
- **License**: Boost Software License 1.0 (permissive — re-distributable with attribution).
- **Coverage**: All Boost.Math special functions and statistical distributions. Test data generated via `boost::math::tools::test_data` at ≥128-bit precision using deliberately naive implementations (Taylor series, recurrence) that share no code with the optimized library — independence is structural.
- **Format**: C++ headers (`libs/math/test/*.ipp`) — large arrays of `{x, expected}` rows.
- **Ingest**: Trivial regex parse of `.ipp` files into JSON. Particularly strong for incomplete beta/gamma, ellint_rj/rd/rf (Carlson forms), expint, hermite, laguerre, jacobi.

### 8. NIST CAVP / ACVP test vectors
- **License**: Public domain.
- **Coverage**: AES, SHA-1/2/3, SHAKE, HMAC, KDF, ECDSA, RSA, DSA, DRBG, KAS-ECC, EdDSA. Each algorithm has KAT (Known Answer Test), MMT (Multi-block Message), MCT (Monte Carlo).
- **Format**: ACVP JSON (modern); legacy CAVS `.rsp` text files. Mirrored at github.com/usnistgov/ACVP-Server and github.com/coruus/nist-testvectors.
- **Ingest**: Direct — already JSON. Drop `testdata/crypto/cavp/<alg>/*.json`. Wire `crypto` package golden tests to read ACVP request schema. **Mandatory** if reality crypto is to claim FIPS-adjacent correctness.

### 9. CORE-MATH project (CeCILL-C / MIT-style mix)
- **License**: CeCILL-C (LGPL-equivalent). Permissive enough for offline reference-value extraction.
- **Coverage**: Correctly-rounded binary32 and binary64 elementary functions: exp, exp2, exp10, log, log2, log10, sin, cos, tan, sinpi, cospi, tanpi, asin, acos, atan, atan2, cbrt, hypot, pow. Distributed with bit-exact reference and exhaustive ulp tests.
- **Format**: C; ulp-comparison driver outputs `(x, f(x), ulp_error)`. Drives glibc/LLVM-libc upstream.
- **Ingest**: Pin `math.Sin`, `math.Cos`, etc. wrappers in reality (where we wrap stdlib for buffered/SIMD variants) against CORE-MATH binary64 outputs at boundary inputs (denormals, π/2 neighborhoods, large reduction).

### 10. PARI/GP test suite
- **License**: GPL-2+ (cannot link/vendor; can observe outputs).
- **Coverage**: Number theory canon — ellinit, ellap (Schoof for #E(F_p)), nfinit (number field), idealfactor, modular forms, L-functions, Stark units, polrootsmod, BSD invariants. Reality slot 320 (ECC) and slot 322 (NIST CAVS) intersect with `ellap` golden values.
- **Format**: GP scripts; testfile-driven regression in pari/src/test/in/ and out/.
- **Ingest**: Run PARI offline, snapshot `(curve, p, ellap_value)` into JSON. Same story as MPFR/Arb (numbers, not code).

### 11. ACM CALGO Algorithm 745 / 757 / 819 / 880 (Special-function specific)
- **License**: ACM CALGO.
- **Coverage**: Alg 715 (Cody SPECFUN reference), Alg 757 (real special-function complements: Carlson, Riemann zeta, exponential integral, Coulomb wave), Alg 880 (complex Voigt/Faddeeva — drives `signal` line-shape), Alg 916 (improved Voigt). Cody's SPECFUN is the historical "ground truth" for IEEE-754 single/double special functions.
- **Format**: Fortran 77/90 with example drivers + expected output text.
- **Ingest**: Same as #3, but called out separately because these are the *most-cited* reference implementations in the floating-point research literature.

### 12. Wolfram Mathematica / Wolfram Alpha
- **License**: Proprietary, but emitted *numeric values* are facts. Inline use in test generation acceptable; redistribution of bulk Mathematica output is gray-area — keep audit trail per value.
- **Coverage**: Universal — N[expr, 50] gives 50-digit reference for almost any expression. Functional gold for items missing from mpmath/Arb (esoteric: Painlevé II, Heun, Mathieu in non-standard form, q-Pochhammer).
- **Format**: Notebook output; export as JSON via `Export["x.json", values]`.
- **Ingest**: Treat as last-resort generator for the long tail. Always cross-check against mpmath when both produce a value — disagreement at digit 15 indicates a Mathematica algorithm switch boundary.

## Pipeline recommendation

**Tier 1 — universal generator (all packages):**
1. Add `tools/golden/` directory.
2. Manifest: `tools/golden/manifest.yaml` enumerating `{package, function, generator, arg_grid, precision_bits, tolerance_ulp}`.
3. Generators: `gen_mpmath.py`, `gen_arb.c`, `gen_mpfr.c`, `gen_xsref_import.py`. Each emits canonical JSON: `{"meta": {...}, "vectors": [{"input": [...], "expected": "0x1.abc...p+3", "tol_ulp": 1}, ...]}`.
4. Hex-float (`%a`) printf format preserves all 53 bits exactly — JSON-safe and unambiguous.
5. Run `make golden` offline (CI nightly, not on every PR) to regenerate. Drift detection: any value change requires SHA-pinned commit message.

**Tier 2 — corpus imports (one-time):**
1. `import_xsref.py` → `testdata/scipy_xsref/<fn>.json` (most coverage for `prob`+`signal`).
2. `import_cavp.py` → `testdata/crypto/cavp/<alg>.json` (mandatory for `crypto`).
3. `import_calgo.py` → `testdata/calgo/alg<NNN>.json` (for Voigt/Bessel).
4. `import_boost.py` → regex-extract `libs/math/test/*.ipp` (broadest special-function coverage).

**Tier 3 — runtime validation:**
1. Each Go package gains `golden_test.go` that walks `testdata/*.json`.
2. Comparison: parse hex-float → float64; assert `Ulp(got, want) <= tol_ulp`. For ball-arithmetic sources (Arb), assert `|got − mid| <= rad`.
3. IEEE-754 edge cases: every corpus must include +Inf, −Inf, NaN, −0, smallest-subnormal, largest-finite as input rows. Failing one is a release-blocker.
4. Tolerance per function (already a stated reality rule) — store in JSON `meta.tol_ulp`.

**Tier 4 — provenance metadata:**
1. Each golden JSON header: `{"meta": {"source": "mpmath-1.4.0", "generator_sha": "...", "generated_at": "2026-05-09T...", "precision_bits": 256, "license": "BSD-3-Clause", "citation": "DLMF §10.2.2"}}`.
2. Hash all golden files at release; embed manifest hash in `reality/version.go`. Detects silent corruption.

**Priority order to implement:**
1. mpmath generator + xsref import (covers 70% of `prob`/`signal`/`physics`).
2. NIST CAVP import (unlocks `crypto` claims).
3. Arb generator (rigorous bounds for ζ/hypergeometric — slots 295/298).
4. Boost.Math `.ipp` regex import (Carlson elliptic, Voigt, expint).
5. CALGO alg-by-alg as needed (long tail).

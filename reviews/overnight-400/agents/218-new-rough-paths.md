## 1. The twenty rough-path primitives

Each entry: capability + reference / composition / LOC / cross-link / blocking-flag.

### Tier-0 — substrate (~340 LOC)

**R1 — `IteratedIntegrals(path [][]float64, depth int) [][]float64`.** Discrete iterated-integral computation via trapezoidal-rule on uniform sampling: `S^k_{i_1,...,i_k}(γ) = ∫_{0<u_1<...<u_k<1} dγ^{i_1}_{u_1} · ... · dγ^{i_k}_{u_k} ≈ Σ_{j_1<...<j_k} Δγ^{i_1}_{j_1} · ... · Δγ^{i_k}_{j_k} · weights`. Returns flattened-tensor slices of shape `[(d^{N+1}-1)/(d-1)]`. Depth 1: just path increment. Depth 2: pairwise increment products in upper-triangle. Higher depth: nested loops. ~140 LOC. **Wholly absent.** Pin against closed-form for piecewise-linear paths: `S^k(γ_lin) = (γ_T - γ_0)^⊗k / k!` (this is Chen's-formula for shuffle-products on linear segments). Reference: Chen-1957 Trans.AMS-89 §1, Lyons-1998 Rev.Mat.Iberoam-14 §2. Cross-language pin: `iisignature.sig(path, N)` byte-equality to 1e-12.

**R2 — `TensorAlgebra` struct + `TensorMul / TensorAdd / TensorScale`.** Layered storage of `T^N(R^d) = ⊕_{k=0}^N (R^d)^⊗k` with per-level `[][]float64` slices and the multiplication `(a ⊗ b)^k = Σ_{i+j=k} a^i ⊗ b^j` (graded tensor product). Internal storage: `levels [][]float64` with `levels[k]` of length `d^k`. ~200 LOC. **Wholly absent.** Pin Chen's relation `S(γ)_{[s,u]} ⊗ S(γ)_{[u,t]} = S(γ)_{[s,t]}` to 1e-10 on random Brownian-bridge paths. Reference: Reutenauer-1993 Free-Lie-Algebras Oxford-Math.Mono §1.4.

### Tier-1 — keystone (~1,200 LOC)

**R3 — `Signature(path [][]float64, depth int) Signature`.** The path signature as element of T^N(R^d). Wraps R1 iterated-integrals into the layered-tensor-algebra structure. For path γ: [0,T] → R^d, returns `(1, ∫dγ, ∫∫dγ⊗dγ, ..., ∫^N dγ⊗...⊗dγ)`. Discrete piecewise-linear signature computed as Chen's-product over segments: `S(γ) = ⊗_{i=0}^{n-1} exp(Δγ_i)` where exp is in the tensor algebra and Δγ_i = γ_{t_{i+1}} - γ_{t_i}. ~280 LOC. **Wholly absent.** Pin against `iisignature.sig` to 1e-12 for d=2,3,4 and N=2,3,4,5 on canonical paths (linear, sinusoidal, Brownian-bridge). Reference: Chen-1957, Lyons-1998, Friz-Victoir-2010 §7.1.

**R4 — `LogSignature(path [][]float64, depth int) LogSignature`.** Log of the signature in tensor algebra: `log S(γ) = Σ_{n≥1} (-1)^{n+1}/n · (S(γ) - 1)^⊗n` (formal log series in T^N). By Chen's theorem, lies in free Lie algebra g((R^d)) ⊂ T((R^d)). Output as coefficients in **Lyndon basis** (R6) of dimension `Σ_{k=1}^N (1/k)·Σ_{j|k} μ(k/j)·d^j` (Witt formula) — strictly smaller than the `(d^{N+1}-1)/(d-1)` flat tensor dimension by factor ~1/N. ~160 LOC + needs R6+R7. **Wholly absent.** Pin against `iisignature.logsig` to 1e-12. Reference: Reutenauer-1993 §1.6, Friz-Hairer-2014 §2.

**R5 — `SignatureKernel(path1, path2 [][]float64) float64`** ⭐ **MOAT.** Computes `k_sig(γ, η) = <S(γ), S(η)>_{T(R^d)}` (untruncated inner product in tensor algebra) via the **Goursat-PDE solution** Salvi-Cass-Foster-Lyons-Lyons-2021 SIAM-J.Math.DataSci-3(3): solve `∂²K/∂s∂t = K · <γ'_s, η'_t>` on `[0, S]×[0, T]` with `K(s, 0) = K(0, t) = 1`. Discretise via finite-difference on the path-grid in `O(L_γ · L_η)` time vs naive truncation cost `O((L_γ + L_η)^N)`. ~280 LOC. **MOAT — no zero-dep Go library ships this; only `sigkernel` PyPI (research-2021) implements it cross-language.** Pin against `sigkernel.SigKernel` to 1e-10. Reference: Salvi-Cass-Foster-Lyons-Lyons-2021, Király-Oberhauser-2019 J.Mach.Learn.Res-20.

**R6 — `LyndonWords(d int, maxLen int) [][]int` / `IsLyndon(word []int) bool` / `LyndonFactorisation(word []int) [][]int`.** Lyndon-1954 / Chen-Fox-Lyndon-1958 enumeration of Lyndon-words over alphabet `{0, ..., d-1}` of length ≤ N via Duval-1988 algorithm in `O(d·n)`. Lyndon-word: strictly less (lexicographic) than all its non-trivial cyclic rotations. Cardinality matches Witt formula `(1/n)·Σ_{k|n} μ(n/k)·d^k`. Critical: log-signature dimension = sum over n=1..N of #Lyndon-words of length n. ~250 LOC. **Wholly absent — placement in `combinatorics/lyndon.go` at the package level (parallel to 217-F4 NonCrossingPartitions in `combinatorics/noncrossing.go`).** Pin: `len(LyndonWords(d, n)) == WittFormula(d, n)` for `(d, n) ∈ {(2,1..6), (3,1..5), (4,1..4)}`. Reference: Reutenauer-1993 §5.1, Duval-1988 J.Algorithms-9, Lothaire-1997 Combinatorics-on-Words §5.

**R7 — `FreeLieProjection(tensor TensorAlgebra) FreeLieElement` / `DynkinIdempotent(level int, d int) [][]float64`.** Reutenauer-1993 §3.1 Dynkin-Specht-Wever idempotent `π_n: T_n(V) → L_n(V)` projects level-n tensor to free-Lie-algebra component via `π_n(x_{i_1} ⊗ ... ⊗ x_{i_n}) = (1/n)·[x_{i_1}, [x_{i_2}, ..., [x_{i_{n-1}}, x_{i_n}]...]]` (right-nested-Lie-bracket). Coefficients in Lyndon basis via Lyndon-factorisation. ~180 LOC. **Wholly absent.** Pin: `π_n` is idempotent (`π_n² = π_n`) and `π_n(L_n) = L_n`. Reference: Dynkin-1947 Mat.Sb-21, Specht-1948 Math.Z-50, Wever-1949 Math.Z-51, Reutenauer-1993 Theorem-3.1.

**R8 — `ChenRelationCheck(path [][]float64, splitIdx int, depth int) (bool, float64)`.** Property-based test: `S(γ)_{[s,t]} = S(γ)_{[s,u]} ⊗ S(γ)_{[u,t]}` for all u ∈ [s, t]. Critical golden-file pin on the multiplicative-functional structure of signatures. ~50 LOC. **Wholly absent.** Pin against synthetic piecewise-linear paths to 1e-12 across split points. ### Tier-2 — high-demand (~1,200 LOC)

**R9 — `ControlledRoughPath` struct `{Y []float64, Yprime []float64}` / `RemainderCheck(...)`.** Gubinelli-2004 J.Funct.Anal-216 controlled rough paths: pair `(Y, Y')` with `Y_t = Y_s + Y'_s · X_{s,t} + R_{s,t}` and `R_{s,t} = O(|t-s|^{2α})` for α ∈ (1/3, 1/2]. Y' is the **Gubinelli-derivative** capturing the leading-order response of Y to X. Verifier checks remainder regularity numerically: `|R| ≤ C·|t-s|^{2α}` over all pairs. ~200 LOC. **Wholly absent.** Pin: for `Y = f(X)` with f smooth, `Y' = f'(X)` and remainder is O(|t-s|^2). Reference: Gubinelli-2004, Friz-Hairer-2014 §4.

**R10 — `RoughDifferentialEquation(f func([]float64, []float64), X RoughPath, Y0 []float64, dt float64) [][]float64`.** Solve RDE `dY = f(Y) dX` by Davie-2008 J.Numer.Anal-17 Picard iteration on truncated-signature: `Y_{t_{n+1}} = Y_{t_n} + f(Y_{t_n}) · X_{t_n, t_{n+1}} + (1/2)·f'(Y_{t_n})·f(Y_{t_n}) · X^{(2)}_{t_n, t_{n+1}}` (level-2 truncation; higher orders extend the formula). ~340 LOC. **Wholly absent.** Pin against Itô-SDE / Stratonovich-SDE via Brownian-path + Lévy-area lift; for additive-noise-RDE `dY = f(Y) dt + σ dX` with X = Brownian, recovers Euler-Maruyama at level-1 and Milstein-like-precision at level-2. **R11 — `ShuffleProduct(a, b TensorAlgebra) TensorAlgebra`.** Ree-1958 shuffle product on T(V) dual to the deconcatenation coproduct: `(x_1...x_p) ⊔⊔ (y_1...y_q) = Σ_{σ ∈ Sh(p,q)} z_{σ^{-1}(1)}...z_{σ^{-1}(p+q)}` summed over `(p, q)`-shuffles. Critical: signatures pair with shuffle-polynomials by integration-by-parts (`<S(γ), a ⊔⊔ b> = <S(γ), a>·<S(γ), b>` — multiplicative pairing on the dual side). ~120 LOC. **Wholly absent.** Pin shuffle-multiplicativity property to 1e-12 on Brownian paths. Reference: Ree-1958 Ann.Math-68, Reutenauer-1993 §1.4.

**R12 — `LevyAreaSample(brownianPath [][]float64, dt float64) [][]float64`.** Sample second-level enhancement of Brownian motion: Lévy-area `A^{ij}_{s,t} = (1/2)(∫_s^t W^i dW^j - ∫_s^t W^j dW^i)` for i ≠ j. **R13 — `MagnusExpansion(A func(float64) [][]float64, t0, t1 float64, depth int, out [][]float64)`.** Magnus-1954 CPAM-7 expansion for matrix-ODE `Y'(t) = A(t)·Y(t)`, `Y(0) = I`: solution is `Y(t) = exp(Ω(t))` with `Ω = Ω_1 + Ω_2 + Ω_3 + ...` graded series in iterated-integrals of Lie-brackets of A. Ω_1 = ∫A; Ω_2 = (1/2)∫∫[A_s, A_u]; Ω_3 = (1/6)∫∫∫([A,[A,A]]+[[A,A],A])-etc. Truncated at depth N gives matrix-ODE-solver of weak-order N+1. ~280 LOC. **Wholly absent — cross-link 205-L8 BCH cross-link.** Pin against closed-form for commuting A (Ω = ∫A); pin against canonical non-commuting test (Pauli-matrix rotation). **R14 — `SignaturePathInversion(sig Signature, depth int, gridLen int) [][]float64`.** Lyons-Xu-2017-Proc.LMS-115(4) reconstruct piecewise-linear path from truncated signature via inversion of the signature-cumulant chain — the kernel of S is exactly tree-equivalence (Hambly-Lyons-2010-Ann.Math-171), so reconstruction recovers the path up to time-reparametrisation and tree-cancellation. ## 4. Risks and gotchas

- **G1. Tensor-algebra storage explosion at high depth/dim.** Storing `T^N(R^d)` requires `Σ_{k=0}^N d^k = (d^{N+1}-1)/(d-1)` floats. For d=10, N=5 that's 111111 floats per signature. Document this prominently; recommend `prob/roughpath/log_signature.go` (Lyndon-basis representation, factor ~1/N smaller) as the production default for d ≥ 5.
- **G2. Numerical instability of signature inversion (R14).** Hambly-Lyons-2010 uniqueness is up to tree-equivalence, so naive inversion of truncated signature has multiple branches. Document loud-fail when input signature has degenerate (close-to-tree) component; provide `IsTreeReduced(path) bool` validator.
- **G3. Stratonovich vs Itô lift choice for Brownian rough paths.** A Brownian path has no canonical second-level enhancement — Stratonovich and Itô lifts differ by `(1/2)·t·δ_{ij}` in the symmetric part of `X^{(2)}`. Document the choice in `LevyAreaSample` (R12); default to Stratonovich (chain-rule-friendly) and offer `LevyAreaSampleIto` as a separate function.
- **G4. Goursat-PDE numerical dispersion (R5).** Salvi-Cass-Foster-Lyons-Lyons-2021 uses centered-finite-differences on a uniform grid; for paths of unequal length, bilinear interpolation introduces second-order error. Document the convergence-rate `O(h^2)` and pin against Romberg-extrapolation reference for parity validation.
- **G5. Magnus convergence radius (R13).** Magnus expansion converges only for `∫₀ᵗ ‖A(s)‖ ds < π` (Hochbruck-Lubich-1997 Numer.Math-78); diverges for high-amplitude long-time-horizon matrix-ODEs. Document loud-fail on `MagnusExpansion` when norm-budget exceeds threshold; recommend RKMK-Lie-group-integrator (slot 204-I13) as the alternative for long-time matrix-ODE.
- **G6. Cubature point count (R16) explodes in dimension.** Lyons-Victoir cubature has `n_N(d) ≈ d^N` points; for d=5, N=5 that's 3125 points per cubature step. Document that cubature is competitive only for `d ≤ 4` and `N ≤ 5`; defer to MLMC (slot 202-S5) for higher-d / higher-order.
- **G7. Lyndon-vs-Hall basis convention.** The free-Lie-algebra has multiple basis conventions: Lyndon (lex-min-rotation), Hall (recursive-Hall-tree), Chen-Fox-Lyndon (slight variant). `iisignature` uses Lyndon; `signatory` uses Lyndon. Document choice as Lyndon (matches both Python references) and ship Hall as an optional alternative if consumer-pull arises.
- **G8. Cross-language byte-equality requires fixing path-discretisation.** `iisignature.sig(path, N)` and reality's `Signature(path, N)` agree only when both use identical-discretisation. Document explicitly the trapezoidal-rule choice and pin the path-grid in golden-file fixtures.

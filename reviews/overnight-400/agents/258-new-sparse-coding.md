**D1 — `dl/sparse_coding_fista.go::SparseCodeFISTA(D, y, lambda, cfg) []float64` ~40 LOC.** Single-call wrapper composing existing `optim/proximal/Fbs(grad, ProxL1, x, work, cfg with Accelerate=true)` for `argmin_α (1/2)||D·α − y||^2 + λ||α||_1`. **D2 — `dl/sparse_coding_omp.go::SparseCodeOMP(D, y, sparsity int) ([]float64, []int)` ~80 LOC.** Single-signal OMP. At iter k pick `j_k = argmax_j |D_j^T·r_{k-1}|`, append to support, solve LS on support via `linalg.QR`-update (incremental QR for support-add), update residual `r_k = y − D_S·α_S`. **D3 — `dl/sparse_coding_batch.go::SparseCodeBatch(D, Y, lambda, mode {FISTA,OMP}) [][]float64` ~80 LOC.** Parallel batch wrapper applying D1 or D2 across N columns of Y. Embarrassingly-parallel via `sync.WaitGroup` over goroutines; each goroutine owns its `work` buffer (no contention). Returns K×N sparse-codes-matrix A as row-major `[][]float64`. Default-batch-size `runtime.NumCPU()`. Refs: standard map-reduce on sparse-coding step from Mairal-Bach-Ponce-Sapiro-2009-§3.

**D4 — `dl/dict_util.go::ColumnNormalize(D []float64, m, K int)` ~40 LOC.** In-place column-l_2-normalize the m×K dictionary matrix. Critical post-D-update step (without it `||d_k||→∞` while `||α||→0` is a free-degeneracy-direction in the joint objective). `ColumnL2Project` variant projects only columns with `||d_k||>1` (allowing within-ball atoms). Refs: Aharon-Elad-Bruckstein-2006-§II.B normalization-step.

**D5 — `dl/dict_update_mod.go::DictionaryUpdateMOD(Y, A [][]float64) [][]float64` ~40 LOC.** Closed-form MOD-step `D = Y·A^T·(A·A^T)^{-1}` followed by D4 column-normalize. Solves the K×K linear system `(A·A^T)·X = (A·Y^T)` via `linalg.Cholesky` (P-d when A has full row-rank — guaranteed when at least K signals are coded with non-empty support and atoms are not collinear; fallback to `linalg.LU` when Cholesky fails-PD-test for diagnostic, error-out for low-rank-A). ### Tier 2 — Batch dictionary update (~520 LOC)

**D6 — `dl/dict_update_ksvd.go::DictionaryUpdateKSVD(Y, A [][]float64) ([][]float64, [][]float64)` ~280 LOC — KEYSTONE.** Aharon-Elad-Bruckstein-2006 atom-by-atom SVD-rank-1 update. For each k=1..K: (1) compute `E_k = Y − Σ_{j≠k} d_j·a_j^T` (rank-1 stripped error), (2) restrict to active-set `ω_k = {i : a_k(i) ≠ 0}` columns of E_k → matrix E_k^R, (3) compute rank-1-SVD of E_k^R via full `linalg.SVD` (or D7 power-iteration when SVD-call expensive), (4) set `d_k ← u_1` (left-singular-vector), `a_k(ω_k) ← σ_1·v_1` (right-singular-vector scaled). Returns updated D AND updated A simultaneously (this is K-SVD's distinguishing feature versus MOD which only updates D). Refs: Aharon-Elad-Bruckstein-2006-IEEE-TSP-54(11). **BLOCKED on `linalg/svd.go`** (~280 LOC, 097-linalg-missing P0 + 215 P0 + this slot's substrate-blocker).

**D7 — `dl/dict_update_aksvd.go::DictionaryUpdateApproxKSVD(Y, A [][]float64, numPower int) ([][]float64, [][]float64)` ~120 LOC.** Rubinstein-Zibulevsky-Elad-2008 approximate-K-SVD using power-iteration in lieu of full-SVD: (1)-(2) as D6, (3') initialize `d_k = E_k^R · a_k(ω_k) / ||...||`, iterate `a_k ← E_k^R^T · d_k`, `d_k ← E_k^R · a_k / ||...||` for `numPower=8` iterations (rank-1-power-method converges geometrically with ratio σ_2/σ_1 — typically converges to relative-1e-6 in 5-10 iterations on natural-image patches). NO SVD dependency. Refs: Rubinstein-Zibulevsky-Elad-2008-Tech-Rep-Technion-CS-2008-08. **READY-TO-SHIP TODAY** — substrate present.

**D8 — `dl/online_dl.go::OnlineDictionaryLearning(streamY <-chan []float64, K int, lambda, batchSize int) (DictStream <-chan [][]float64)` ~280 LOC — SOTA-2010s-2020s SINGULAR-MOAT.** Mairal-Bach-Ponce-Sapiro-2009 streaming DL via accumulator matrices `A_t = (1−η)·A_{t−1} + α_t·α_t^T` and `B_t = (1−η)·B_{t−1} + α_t·y_t^T`.
